(識別コード) (int型の数値1), (int型の数値2), (int型の数値3), (int型の数値4), (int型の数値5)

-------------------------------------------------

注意事項
bitmapファイルの最後に必ず改行を入れて下さい

-------------------------------------------------

(識別コード)
B：ブロック1つ分の情報
C：ブロックの種類に対応する色

-------------------------------------------------

(int型の数値1)
Bの場合：ブロックの座標x
Cの場合：ブロックの赤色(0～255)

-------------------------------------------------

(int型の数値2)
Bの場合：ブロックの座標y
Cの場合：ブロックの青色(0～255)

-------------------------------------------------

(int型の数値3)
Bの場合：ブロックの座標z
Cの場合：ブロックの緑色(0～255)

-------------------------------------------------

(int型の数値4)
Bの場合：ブロックの種類(下記参照)
Cの場合：ブロックの透明色(0～255)

-------------------------------------------------

ブロックの種類
何も置かない：-1
動かせるブロック：0
動かせないブロック：1
ゴールブロック：2
プレイヤーのスタート位置(描画は無し)：3
スイッチ：4
ドア：5
ワープブロック(ワープ未設置)：6
ワープブロック(ワープ設置済)：7
その他のギミック:8～(プログラム上に記述は無いから追加する際は教えて)

-------------------------------------------------

(int型の数値5)
Bの場合：同じ番号の場合、隣接に関係なく繋がる
Cの場合：記述する必要なし

------------------以下書き方の例------------------

B 0, 0, 0, 0, 0
B 2, 0, 1, 1, 1

C 0, 0, 0, 255
C 255, 255, 255, 255
